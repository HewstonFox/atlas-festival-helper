Implementation Notes

Content Script Usage:

The schedule content script now uses the shared i18n utility instead of calling chrome.i18n.getMessage() directly.

Example: i18n.getMessageWithFallback('buildSchedule', 'Build Schedule')

Shared i18n Utility:

The extension uses a shared shared/i18n.js utility that provides:

  • i18n.getMessage(key) - Get message without fallback
  • i18n.getMessageWithFallback(key, fallback) - Get message with fallback (recommended for content scripts)
  • i18n.initialize() - Initialize for popup/options pages
Fallback Strategy:

All translations use getMessageWithFallback() which provides English fallback text for robustness.

Dynamic Content:

Button text, tooltips, and alert messages are all internationalized and will display in Ukrainian by default.
